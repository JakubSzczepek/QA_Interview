Kontekst

SkyBook to REST API systemu rezerwacji lotów i zarządzania biletami lotniczymi. API obsługuje:

  • Zarządzanie lotami (CRUD)
  • Rezerwacje z pasażerami, miejscami i płatnościami
  • Baza lotnisk
  • Przeliczanie walut

Twoje zadanie

  1. Zapoznaj się z API korzystając z dostępnych narzędzi
  2. Przetestuj API pod kątem poprawności działania — szczególnie pod kątem poniższych reguł biznesowych
  3. Zaraportuj znalezione błędy
💡 Wskazówka: REST Client ma 12 gotowych requestów w panelu bocznym — możesz je wysyłać od razu lub modyfikować. Nie musisz pisać JSON od zera.

BR-03Segment: departure/arrivalairportCode – IATA 3 litery; dateTime – ISO 8601; arrival po departure; durationMinutes > 0
BR-04Segment: aircraftObiekt wymagany; model 1–100 znaków
BR-05Cena bazowa i walutabaseFare ≥ 0.00, max 2 dec; currency ∈ {USD, EUR, GBP, PLN}

Context

SkyBook is a REST API for a flight booking and airline ticketing system. The API supports:

  • Flight management (CRUD)
  • Bookings with passengers, seats and payments
  • Airport database
  • Currency conversion

Your task

  1. Explore the API using the available tools
  2. Test the API for correctness — especially against the business rules below
  3. Report the bugs you find
💡 Tip: REST Client has 12 ready-to-use requests in the sidebar — you can send them immediately or modify them. No need to write JSON from scratch.

Business Rules

#RuleDescription
BR-01Flight numberRequired, unique, format: 2–3 letters + - + 1–5 digits (e.g. SB-1234)
BR-02Flight segmentsMin. 1 required; direct flight = 1 segment, connecting = 2+
BR-03Segment: departure/arrivalairportCode – IATA 3 letters; dateTime – ISO 8601; arrival after departure; durationMinutes > 0
BR-04Segment: aircraftObject required; model 1–100 characters
BR-05Base fare and currencybaseFare ≥ 0.00, max 2 dec; currency ∈ {USD, EUR, GBP, PLN}
BR-06TaxesOptional; code 2 uppercase letters, unique; amount ≥ 0.00
BR-07FeesOptional; feeCode unique; amount ≥ 0.00; max 10 fees
BR-08Total priceAuto: baseFare + Σtaxes + Σfees − discount; value must be correct
BR-09DiscountOptional; percentage 0–100; validUntil not in the past
BR-10Available seatsDecreased on booking, restored on cancellation
BR-11PassengersMin. 1, max 9 per booking
BR-12Passenger datafirstName/lastName 1–100 chars; dateOfBirth ISO; valid email; international phone
BR-13Seat assignmentseatNumber unique per flight; class ∈ {ECONOMY, BUSINESS, FIRST}; type ∈ {WINDOW, MIDDLE, AISLE}
BR-14PaymentRequired; method ∈ {CREDIT_CARD, DEBIT_CARD, BANK_TRANSFER, BLIK}; cardDetails required for cards
BR-15Booking statusPENDING → CONFIRMED, PENDING → CANCELLED, CONFIRMED → CANCELLED; no reversal
BR-16Flight deletionCannot delete a flight with bookings
BR-17Currency conversionCorrect conversion between USD, EUR, GBP, PLN
BR-18PaginationDefault: page=0, size=10, max size=100
BR-19Flight searchCase-insensitive, partial match
BR-20HTTP Status CodesPOST → 201, DELETE → 204, Not Found → 404, Validation → 400
